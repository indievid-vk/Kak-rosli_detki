## 1. Стек технологий
- **Фреймворк:** React 18 + TypeScript + Vite.
- **Стилизация:** Tailwind CSS + shadcn/ui.
- **Хранилище данных:** IndexedDB (библиотека `idb` для удобной работы через Promise).
- **Формат приложения:** PWA (Progressive Web App) с полной автономностью без backend-сервера.

---

## 2. Архитектура базы данных (Локальное хранение)

Приложение полностью Serverless (Zero Backend). Все данные пользователя живут исключительно в браузере (IndexedDB).

### Описание схемы БД (`src/lib/db.ts`)
БД проинициализирована под именем `BabyAppMedia` и содержит три объектных хранилища (Object Stores):

1. **`children`** — Список профилей детей.
   - `id` (UUID)
   - `firstName`, `lastName`, `birthDate`, `photoUrl` (base64 строка или ссылка), `createdAt`.
2. **`records`** — Записи дневника (события, словарики, достижения).
   - `id` (UUID)
   - `childId` — привязка к ребенку.
   - `title`, `description`, `videoUrl` / `audioUrl` — текст записи и медиа.
3. **`media`** — Бинарные/Base64 данные больших файлов (аудио, видео, большие фото).
   - *Ключ-значение.* Ключ — кастомный ID (`audio_123`, `media_456`), значение — base64 data URL.

### Ограничения медиа:
* В PWA память браузера ограничена (от 500 Мб до пары гигабайт в зависимости от устройства).
* **Сработавшее ограничение:** При попытке загрузить свыше 50–100 Мб в одной записи, браузер (особенно мобильный Safari/Chrome) уходит в "Out of Memory" (Опаньки!).
* **Решение:** Везде (в компонентах загрузки `Home.tsx`, `ChildProfile.tsx` и окне информации `BackupFooter.tsx`) установлены жесткие валидаторы на `<input type="file">` — `file.size > 50 * 1024 * 1024` (50 Мб). Если файл больше, происходит `return` с предупреждающим `alert()`.

## 3. Резервное копирование (Export / Import)

Поскольку IndexedDB сбрасывается при очистке кэша браузера, реализован механизм дампа данных в `JSON` файл.

### Экспорт (Backup)
* **Процесс:** Приложение собирает массивы `children`, `records` и `media`.
* **Избежание падений памяти:** Файл может весить очень много. Поэтому он собирается в массив строк (chunks) `chunks = ['{"children":', JSON.stringify(...), ...]` вместо склейки одной огромной строки. Затем используется `new Blob(chunks)`.
* Скачивание файла реализуется через виртуальный элемент `<a>` — `URL.createObjectURL(blob)`.

### Импорт (Restore)
* Парсинг загруженного JSON файла.
* Запись данных напрямую через `transaction('readwrite')` циклами в `children`, `records` и `media`.

## 4. Настройка PWA архитектуры для GitHub Pages

Чтобы проект мог быть установлен на телефон как нативное приложение через иконку "На экран Домой", используется связка: `manifest.json`, `sw.js` и `pwa-setup.js`.

### 4.1. Корректный Base URL (Subfolder Routing)
При хостинге на GitHub Pages приложение живёт в подпапке (напр. `https://username.github.io/Kak-rosli-detki/`).
В `vite.config.ts` обязательно настраивается:
```typescript
base: mode === 'production' ? '/Kak-rosli-detki/' : '/'
```

### 4.2. Файл `manifest.json` (/public/manifest.json)
Обязательные поля:
- `"start_url": "./"` (точка предотвращает потерю subfolder-структуры URL).
- `"display": "standalone"` (убирает рамку браузера).
- `"icons"` (массив иконок 192x192 и 512x512).

### 4.3. Service Worker (`/public/sw.js`)
Service Worker (SW) кеширует файлы для оффлайн работы. 
**Важнейший нюанс для GitHub Pages:** Все пути в массиве кеша должны быть относительными:
```javascript
const urlsToCache = [
  './', 
  'index.html',
  'manifest.json',
  'pwa-setup.js'
];
```
В событии `fetch` (сетевые запросы) SW перехватывает роутинг и, если сеть недоступна, возвращает `/` или `./` (для react-router корректной работы).

### 4.4. Регистрация SW (`/public/pwa-setup.js`)
```javascript
navigator.serviceWorker.register('sw.js') 
```
Обратите внимание, `sw.js` вызывается *без* начального слеша `/`, что гарантирует его корректное разрешение внутри текущего репозитория (`/Kak-rosli-detki/sw.js`). В `index.html` этот скрипт вызывается просто как `<script src="pwa-setup.js"></script>`.

## 5. UI/UX решения (Дизайн)

- **Диалоговые окна и фокус:**
  В длинных окнах (например, о приложении или резервном копировании) использовались хуки `useRef` и `useEffect` для авто-скроллинга вверх при открытии окна:
  ```typescript
  const scrollRef = useRef<HTMLDivElement>(null);
  useEffect(() => {
    if (isOpen && scrollRef.current) scrollRef.current.scrollTop = 0;
  }, [isOpen]);
  ```
  Иногда модальные окна перехватывают фокус или имеют скроллинг баги; внедрение невидимо-сфокусированного элемента `<div tabIndex={0} autoFocus />` в начало скроллируемого контейнера помогает стабилизировать отображение.

- **Стилизация под iOS:**
  Использование `apple-touch-icon` в `<head>` документа `index.html`, чтобы иконка приложения на экранах iPhone выглядела безупречно.

## 6. Как перенести этот опыт в другие проекты (напр. "Путь спасения")

1. **Базовая структура Vite:** Используйте абсолютно те же настройки `vite.config.ts`, изменяя лишь `base: '/repo-name/'`.
2. **PWA комплект:** Просто скопируйте файлы `sw.js`, `pwa-setup.js` и `manifest.json` в корневую папку `/public` нового проекта. В `index.html` пропишите `manifest` и скрипт `pwa-setup.js` с относительными путями (без слешей). Обновите `start_url` на `./`.
3. **Хранение данных:** Если проект требует оффлайн хранения, перенесите логику библиотеки `idb` (`src/lib/db.ts`). IndexedDB масштабируется под любые задачи (как медиа, так и JSON-документы), при этом важно валидировать размеры файлов, как мы это сделали здесь.
